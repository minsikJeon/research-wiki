# 3DFlowAction

## One-line summary

[[im2flow2act]] lifted to 3D: 4-channel (u, v, z, visibility) flow
generated by an AnimateDiff world model trained on **ManiFlow-110k**
(110K cross-source 3D flows). Action extraction is closed-loop via
GPT-4o verification + IK-aware grasp selection + optimization policy
(no action labels).

## Inputs / outputs

- **Training inputs:** ManiFlow-110k (BridgeV2/RT-1/AgiWorld/DROID/
  RH20T-Human/HOI4D/LIBERO; 110K 3D flow trajectories).
- **Inference inputs:** initial RGBD scene, language task instruction.
- **Inference outputs:** sequence of SE(3) end-effector poses.

## How it works

### 1. ManiFlow-110k construction

- Grounding-SAM2 → gripper mask from frame 1.
- Sample grid points outside gripper → run [[cotracker3]] → keep
  motion points → max bbox = moving object region.
- Co-Tracker3 again on object region → 2D flow.
- DepthAnythingV2 → per-frame depth → project 2D flow to 3D.
- Optional camera motion removal.

### 2. 3D Flow World Model

- SD v1.5 backbone with LoRA + motion modules (AnimateDiff style).
- **Bypass the SD VAE** because it can't encode depth — feed
  4-channel 3D flow directly to UNet.
- Conditioning: CLIP-encoded initial RGB + task prompt; sinusoidal PE
  for initial points F₀.

### 3. Closed-loop flow generation

`T = SVD(P₂, P₁)` (first ↔ last flow points). Apply T to object point
cloud → re-render predicted final RGB scene → GPT-4o checks alignment
with instruction → re-generate if FAIL.

### 4. Task-aware grasp pose

GPT-4o suggests grasp region → AnyGrasp candidates → apply T → IK
check reachability → select reachable, task-relevant pose.

### 5. Optimization action policy

N farthest-point samples on object. At each timestep t, minimize 3D
distance between current end-effector-induced object pose and
predicted-flow position. No action labels needed.

## Where it's been applied

[[zhi-2025-3dflowaction]] — Dobot XTrainer + Franka on
pour-tea, insert-pen, hang-cup, open-drawer tasks. 70% average SR
vs. 20–25% for AVDC / ReKep / Im2Flow2Act* baselines.

## Known limitations

- **Optimization policy is slow + reactive only via re-plan loop** —
  no within-step reactivity.
- **Depth from per-frame DepthAnythingV2** → noisy; Dex4D's
  median-rescaled relative-depth is smoother.
- **GPT-4o verifier latency** (~seconds) — not suitable for highly
  dynamic tasks.
- **Parallel-jaw only** — no dexterous hands.

## Related methods

- **[[im2flow2act]]:** direct 2D ancestor — same AnimateDiff +
  flow-as-interface pattern.
- **[[track2act]]:** SVD-fit transformation is the same pattern as
  Track2Act's per-step PnP rigid-transform; 3DFlowAction does it once
  end-to-end and verifies with GPT-4o.
- **[[dex4d]]:** parallel descendant that swaps optimization policy
  for sim-to-real RL on dexterous hand; relative-depth calibration
  improves over per-frame depth.
- **[[cotracker3]]:** label generation tracker.
