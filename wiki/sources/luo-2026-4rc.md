# 4RC: 4D Reconstruction via Conditional Querying Anytime and Anywhere

## TL;DR

4RC ("ARC") is an **encode-once, query-anywhere-and-anytime** feed-forward
4D reconstruction model. A transformer encodes the whole video into a
**compact 4D latent**; a conditional decoder then queries that latent
for **any source frame × any target time**, emitting base geometry +
relative 3D motion. **Minimally factorized:** each frame =
viewpoint-invariant base geometry + time-dependent relative motion.
NTU S-Lab + Oxford VGG. Compares favorably against [[trace-anything]],
[[any4d]], [[v-dpm]], ST4RTrack.

## Why it matters

The **query-based decoding** paradigm is the key differentiator. Other
feed-forward 4D methods either output everything at once per frame (V-DPM,
Any4D) or decode trajectories per pixel (Trace Anything). 4RC's
"encode-once / decode-on-demand" maps directly to **inference-time
flexibility** — you only pay for what you ask.

For robotics use cases that need motion only at certain timestamps (e.g.
predicting future scene state at t+1, t+2 but not all intermediate
timesteps), this is the architecturally right design.

## Key claims

- **Unified feed-forward 4D from monocular video.** No optimization
  loop, no auxiliary estimators. Single network.
- **Encode-once, query-anywhere/anytime.** Compact 4D latent serves as
  the centralized representation; decoder queries it conditioned on
  source frame + target time.
- **Minimally factorized 4D representation:** for query frame `I_i` at
  target time `τ`:
  - `P^{t_i}_i` — base 3D geometry at source time.
  - `ΔP^{t_i → τ}_i` — 3D displacement from source time to target.
  - Total: `P^{t_i → τ}_i = P^{t_i}_i + ΔP^{t_i → τ}_i`.
- **Leverages monocular geometry pretraining** ([[depth-anything-3]]
  is cited as a foundation for the base-geometry branch).
- **Beats prior + concurrent methods** across 4D tasks: camera pose,
  video depth, point cloud reconstruction, 3D point tracking, dense
  motion modeling.
- **Critiques concurrents directly:** [[v-dpm]] = "inflexible decoding +
  high cost"; [[trace-anything]] = "Bézier curves struggle with
  high-frequency dynamics"; [[any4d]] = "only predicts scene flow from
  the first frame, can't query motion from arbitrary viewpoints."

## Methods

- **Encoder:** ViT patchify all video frames + ⊕ camera tokens → run
  through ViT encoder. Output: tokens `F_i` per frame and per-frame
  camera tokens `C_i`.
- **Decoder (conditional query):** for each query `(i, τ)`:
  - Combine `F_i` with target-time token `T̂_τ` via AdaLN.
  - K decoder blocks attend across spatial + temporal context.
  - Geometry head produces depth, ray, camera; motion head produces
    `ΔP^{t_i → τ}_i`.
- **Training:** monocular video sequences with 4D ground truth.
- **Backbone:** standard ViT; explicitly leverages monocular geometry
  representation (Lin et al. 2025 = [[depth-anything-3]]).

## Results (headline)

- Competitive or SOTA across camera pose estimation, video depth
  prediction, point cloud reconstruction, **3D point tracking**, and
  dense motion modeling.
- Explicitly beats [[v-dpm]] (high cost, inflexible decoding) and
  matches or beats [[any4d]] on most metrics.

## Limitations / open questions

- "Compact 4D latent" capacity may bottleneck for very long videos —
  no scaling study reported.
- The motion factorization (base + displacement) may not generalize to
  motions with significant geometry change (e.g., topology change,
  appearing/disappearing objects).
- Like all feed-forward 4D methods in this batch, real-world dynamic
  evaluation outside curated benchmarks is limited.
